Mini DMP (20 pts):
For this project, I will analyze trends in open-source activity on GitHub's public
Application Programming Interfaces. Since the datasets included in the previous Data Plan
Management assignment do not contain an API, I will use scenario C (GitHub Repository
Analytics) for this assignment to analyze open-source project trends and activity. The purpose of
this assignment is to design and utilize an artificial intelligence collection agent to gather,
manage, and process GitHub repository analytics effectively and ethically.
The primary objective of the project is to assess and analyze activity patterns across
several public repositories. This would include the following: measuring repository growth and
changes; tracking contributor engagement (commits, editing, the number of physical
contributors); looking at commit activity over time to understand individual and group processes
while understanding code development; comparing repositories across various topics. This
would allow us to apply basic statistics to analyze data and repository structure, but ultimately to
apply API fundamentals, ethical data management, and quality AI reporting.
Data sources include repository information, contributor activity, issue counts, and
commit activity (code, text, etc.). The focus of this will be on using R, as I am much more
familiar with the programming language, using the sample code provided in the assignment
repository. Examples of data sources include the “GitHub Rest API” and the “GitHub GraphQL

The following variables may be collected and analyzed:

● Contributor activity: contributor names or usernames (as displayed in GitHub); total
number of commits and contributions; frequency.
● Pull requests: counts; status (open or closed); resolution time.
● Temporal variables: commit dates; open or closing dates; and trending information.
